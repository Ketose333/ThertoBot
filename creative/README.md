# creative 운영 가이드

이 폴더는 톤/목적 혼선을 막기 위해 콘텐츠 성격별로 분리해 관리한다.

## 디렉터리 구조
- `game/`: 게임 구현 전제 문서(서사 + 구현 스펙)
- `non_game/`: 게임 구현 목적이 아닌 단일 서사/콘셉트 문서
- `novel/`: 장편 소설 운영/본문 문서

## novel 운영 기준(루트 안내)
- 실행 단일 기준: `novel/novel_operation_guide.md`
- 세계관 고정값: `novel/novel_story_bible.md`
- 에피소드 상위 아웃라인: `novel/novel_episode_outline.md`
- 에피소드 적용 맵(1~4): `novel/novel_episode_outline_ep1_ep4.md`
- 연재 전략: `novel/novel_release_strategy.md`
- 점검/수정 로그: `novel/novel_review_log.md`
- 연속성 점검표: `novel/novel_rewrite_continuity_check_ep1_ep4.md`
- 통합 리라이트 본문: `novel/stay_awake_full_rewrite.md`

## 파일명 규칙 (공통)
- 자연어 기반의 짧은 영어 파일명 사용
- `snake_case` 사용
- 확장자는 `.md` 고정
- 단어 수는 최대 4개(`_` 기준 분리)

## 권장 패턴
- `[topic]_[doc_type].md`

## 금지
- 날짜/메타데이터가 과하게 붙은 긴 파일명
- 4단어 초과 파일명
- `final_final_really_final.md` 같은 모호한 파일명

## 점검/수정 출력 규칙 (creative 공통)
- 소설/서사 수정 제안은 기본적으로 **수정 전 단락 ↔ 수정 후 단락 1:1 대응** 형식으로 작성한다.
- 제안 문장은 기존 본문에 바로 대체 가능한 수준(복붙 가능)으로 제공한다.
