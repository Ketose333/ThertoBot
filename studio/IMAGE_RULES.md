# IMAGE_RULES.md

한태율 이미지 생성의 단일 규칙 소스.
프롬프트 가드는 이 문서 섹션만 사용한다.

## COMMON_IDENTITY_LOCK
- reference image와 동일 인물/캐릭터 정체성을 유지한다.
- 얼굴 비율·헤어라인·핵심 인상은 고정하고, 포즈/표정/의상/장면 요소만 변형한다.
- 정체성은 유지하되 표정/무드/디테일은 요청에 맞게 유연 반영한다.

## REF_IMAGE_POLICY
- 생성 결과물 경로(`~/.openclaw/media/image/*`)는 ref-image로 사용 금지.
- 아바타/원본 기준 이미지만 ref로 사용.

## REAL_STYLE_GUARD
- 실사(photorealistic) 톤 유지, 2D/카툰/셀셰이딩 금지.
- 과한 색보정/필터/글리치/렌즈왜곡 금지.
- 구도 고정 문구를 강제하지 않는다(요청 장면 우선).

## TWO_D_STYLE_GUARD
- 2D/chibi 스타일 허용.
- 2D 정체성은 디스코드 봇 프사 기준으로 일치.

## FRAMING_AND_POSE_BASELINE
- 기본 프레이밍은 흉상~상반신 클로즈업.
- 인물 화면 점유율 60~75%를 기본으로 유지.
- 얼굴 각도/시선은 요청 장면에 맞춰 자연스럽게 유지한다.

## BACKGROUND_QUALITY_BASELINE
- 읽히는 텍스트/로고/브랜드 표시는 기본 금지.
- 장면은 과밀 소품/과장 장식/복잡 패턴 누적을 피한다.
- 실내/야외 모두 피사체 집중을 해치지 않게 배경 복잡도를 낮춘다.
- 실내 장면에서 우드톤(원목 벽/마루/우드 패널/우드 가구 메인 톤)은 기본 금지.
- 우드톤이 메인으로 검출되면 실패로 간주하고 재생성한다.

## NANO_BANANA_PRO_GUARD
- `nano-banana-pro-preview` 사용 시 reference 일치도를 최우선한다.
- 요청 없는 성별/인종/연령/얼굴구조 변화 금지.

## HARD_CASE_AVOIDANCE
- 고난도 패턴 의상, 반사·투명·발광 소재, 얼굴 가림 포즈는 기본 회피.
- 기본 장면은 장식 적은 집 실내 무드 우선.
- 사용자가 야외/특정 장소를 명시하면 해당 장소를 우선 적용(실내 기본값보다 우선).
- 야외일 때는 군중·네온 간판 다량·복잡 텍스트 배경을 가급적 회피.
- 전신보다 상반신/흉상 구도를 우선해 왜곡 리스크를 줄인다.

## OUTFIT_ONLY_LOCK
- 의상 변경 요청에서는 얼굴/헤어/비율/정체성 절대 변경 금지.
- 변경 범위는 의상/소품/텍스처/색상으로 제한.
- 데님 메인 코디는 사용자 명시 요청 없으면 회피.
- 저지/슬럽 코튼류 티셔츠 질감은 사용자 명시 요청 없으면 회피.

## PROMPT_INPUT_POLICY
- 사용자 프롬프트는 핵심 변화(표정/포즈/장면)만 짧게 입력해도 된다.
- `한태율 실사 인물 사진` 같은 식별 문구는 생략해도 ref + 규칙으로 보완한다.
- 입력이 짧으면 기본값을, 입력이 구체적이면 요청 조건을 우선 반영한다.

## PROMPT_MINIMALISM
- 프롬프트는 '정체성 고정 + 핵심 장면 + 금지사항' 중심으로 최소화.
- 같은 의미 수식어 반복(감성/무드/분위기 중복)은 피한다.
- 추가 조건은 1~2개씩 증분 적용해 과적합을 줄인다.

## USER_VISUAL_HINT_KETOSE
- 승세(케토스) 생성 시 기본 인상: 백금발, 큰 눈, 귀엽고 포근한 분위기.
- 의상은 흰 셔츠 고정 아님(다양한 캐주얼 허용).
