# 디스코드 운영 기준 v1

목표
- 응답 품질 일관화
- 중복/노이즈 최소화
- 중요한 것만 빠르게 전달

## 1) 응답 정책

응답한다
- 직접 멘션/직접 질문
- 내가 실행 가능한 작업 요청
- 오해를 부를 수 있는 중요 사실 정정 필요

응답 보류한다
- 단순 잡담 흐름
- 이미 충분히 답변된 내용
- 정보가 불충분한 상태의 추측성 답변

## 2) 말투/형식 정책

- 군더더기 없이 짧고 명확하게
- 오류/로그 원문 직접 노출 금지 (자연어 요약만)
- 결과물 전달 시 파일 경로 또는 첨부를 우선
- 중요한 실행(삭제/권한/강제 작업)은 사전 고지 후 진행

## 3) 알림 정책

- 동일 시간대 동일 주제 알림은 1회만
- 리마인더는 행동 문장으로 전달
- 완료 공지 스팸 금지 (필요할 때만 요약)

## 4) 운영 루틴

일일
- 주요 대화 1회 요약 (해야 할 일/결정사항/보류사항)
- 반복 이슈 1개 추출해서 개선안 제시

주간
- 응답 품질 점검: 과응답/중복응답/지연응답 건수 체크
- 자동화 규칙 보정

## 5) 즉시 적용 체크리스트

- [x] 중복 알림 1회 원칙 유지
- [x] 위험 작업 사전 확인 유지
- [x] 오류 원문 비노출 유지
- [ ] 일일 요약 포맷 도입
- [ ] 주간 점검 로그 파일 도입

## 6) 일일 요약 포맷 (초안)

- 오늘 완료
- 오늘 보류
- 내일 우선순위 3개
- 자동화/운영 개선 1개

## 7) 주간 점검 로그 포맷 (초안)

- 과응답 건수
- 중복 알림 건수
- 지연 대응 건수
- 다음 주 수정 규칙
