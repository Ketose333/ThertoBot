# 채널 맥락 위생 설계 v1

목표
- 채널 간 맥락 단절/오염 동시 방지
- 메시지 복제 없이(=동기화 미러링 없음) 공통 작업 상태만 공유
- 필요할 때만 맥락 주입(기본은 최소 맥락)

## 1) 4층 방어 구조

1. 입력 필터
- 시스템 알림(heartbeat/cron 메타)은 작업 맥락에서 제외
- 단순 반응/이모지 테스트는 업무 맥락으로 승격 금지

2. 채널별 작업 메모
- `memory/channels/<channel>.md`에 채널별 진행상황만 보관
- 개인 선호/전역 운영 규칙은 `MEMORY.md`에만 유지

3. 전역 상태 요약
- 채널 공통 상태(현재 프로젝트, 결정사항, TODO)는 `memory/global-context.md`에 짧게 유지
- 채널 응답 전: 전역 상태 + 현재 채널 메모만 참조

4. 오염 차단 규칙
- 톤/캐릭터/모델 규칙은 전역 고정값만 사용
- 다른 채널의 감정선/농담/일회성 지시는 자동 이월 금지

## 2) 운영 규칙

- 기본 응답 소스 우선순위
  1) 현재 메시지
  2) 현재 채널 메모
  3) 전역 상태 요약
  4) 장기 메모(MEMORY.md)

- 채널 전환 시
  - 직전 채널 맥락을 그대로 들고 가지 않고, 전역 상태만 재주입

- 큰 작업 종료 시
  - 결과를 `global-context.md`에 3줄 이내로 업데이트
  - 채널별 상세는 해당 채널 메모에만 남김

## 3) MCP 연동 방안(준비)

- MCP 사용 가능 시 우선 적용할 것
  - 채널 메타 저장소(채널ID-프로젝트 매핑)
  - 대화 임베딩 검색(채널 범위 필터 필수)
  - 정책 룰엔진(시스템/유저/테스트 메시지 분류)

- MCP 적용 원칙
  - "검색 범위 제한"이 기본값(현재 채널 우선)
  - 전역 병합은 명시적 요청 있을 때만

## 4) 금지

- 채널 메시지 자동 미러링으로 맥락 공유 해결하려고 하지 않음
- 과거 로그 대량 재주입 금지
- 불분명한 지시는 전역 규칙 변경으로 승격 금지
